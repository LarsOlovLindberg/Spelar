Att göra – go-live (checklista)

Mål: imorgon ska du kunna verifiera att live-credentials fungerar (utan att lägga ordrar), och därefter kunna “arma” live kontrollerat.

TOP 3 (snabbt imorgon)

  1. Öppna Live readiness och verifiera att LIVE-ålder sjunker.
  2. På VPS: sätt TRADING_MODE=live + creds, men håll PM_MAX_ORDERS_PER_TICK=0 (0 risk). Restart.
  3. Kör sync+deploy data-only och kontrollera att pm_open_orders.json blir ok:true på webben.

Om du bara gör dessa tre så vet du om live-länken fungerar, utan att kunna lägga en order.

0) Snabb sanity (kedjan)

  • Öppna Live readiness och tryck Ladda om.
  • Verifiera att LIVE-ålder minskar och att filerna finns (inga 404).
  • Verifiera att paper-saldo syns på Översikt (då vet du att pipeline VPS → sync → web/data → portal fungerar).

1) Förbered “live men 0 risk”

Syftet här är bara att se att privata endpoints fungerar. Vi blockerar order helt.

  • SSH in: ssh root@77.42.42.124
  • Editera env: nano /etc/spelar-agent.env
  • Sätt/ändra (utan att posta secrets någonstans):
  • TRADING_MODE=live
  • POLY_LIVE_CONFIRM=YES
  • POLY_CHAIN_ID=137
  • POLY_PRIVATE_KEY=<din_private_key>
  • POLY_CLOB_API_KEY=<din_api_key>
  • POLY_CLOB_SECRET=<din_api_secret>
  • POLY_CLOB_API_PASS_PHRASE=<din_passphrase>

2) Blockera order helt (viktigt)

  • Se till att följande finns i /etc/spelar-agent.env (för “0 risk”):
  • PM_MAX_ORDERS_PER_TICK=0
  • EDGE_THRESHOLD=1.0 (så inget edge kan trigga)
  • PM_ORDER_SIZE_SHARES=1 (spelar ingen roll när max=0, men håll låg)
  • Valfritt extra skydd: skapa killswitch-filen: touch /opt/spelar_eu/vps/out/KILLSWITCH

3) Restart + verifiera på VPS (utan deploy först)

  • Restart: systemctl restart spelar-agent
  • Status: systemctl --no-pager --full status spelar-agent | head -n 40
  • Verifiera snapshots lokalt på VPS:
  • cat /opt/spelar_eu/vps/out/pm_open_orders.json → ska visa ok:true om live-client är korrekt konfigurerad.
  • cat /opt/spelar_eu/vps/out/kraken_futures_private.json → ok:true endast om Kraken keys är installerade.

4) Sync + deploy data-only (publicera status till portalen)

  • På din Windows-maskin (repo-rot):
  • .\scripts\sync-vps-stats.ps1 -HostName "root@77.42.42.124" -DeployDataOnly -FtpConfigPath .\ftp_config.local.json
  • Verifiera i webben:
  • /data/pm_open_orders.json → ok:true (eller tydligt felmeddelande).
  • /data/kraken_futures_private.json → ok:true eller keys_missing.

5) När du vill börja lägga riktiga ordrar (kontrollerat)

  • Först: ta bort killswitch om den finns: rm -f /opt/spelar_eu/vps/out/KILLSWITCH
  • Ändra i /etc/spelar-agent.env:
  • PM_MAX_ORDERS_PER_TICK=1
  • EDGE_THRESHOLD=0.02 (eller vad du vill köra)
  • Restart: systemctl restart spelar-agent
  • Följ upp i portalen på pm orders och Execution log.

Tips: håll första passet extremt konservativt (max 1 order/tick, liten storlek) och verifiera alltid att loggarna ser rimliga ut innan du ökar.
